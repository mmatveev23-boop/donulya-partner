# ДОНУЛЯ — Логика партнёрской воронки (полный автомат)

## Общая схема

```
ПАРТНЁР                           КЛИЕНТ
   │                                 │
   ▼                                 │
v2.html (обучение 15 экранов)        │
   │                                 │
   ▼                                 │
Кнопка "Перейти в группу"            │
   │                                 │
   ▼                                 │
Мессенджер (TG/VK/MAX)              │
Бот приветствует:                    │
"Добро пожаловать! Ваша              │
 реферальная ссылка:                 │
 donulya.ru/ref?ref=IVAN123"         │
   │                                 │
   ├─── Канал 1: прямой контакт      │
   │    Партнёр пишет боту:          │
   │    "Иван +79121234567"          │
   │    Бот парсит → Google Sheet    │
   │    → уведомление партнёру       │
   │                                 │
   └─── Канал 2: реф. ссылка ───────▶│
        Партнёр раскидывает           │
        ссылку в группах/чатах        ▼
                                  ref.html?ref=IVAN123
                                  (конверсионный лендинг)
                                     │
                                     ▼
                                  Шаг 1: Имя + Телефон + Долг
                                     │
                                     ▼
                                  Шаг 2: Выбор мессенджера
                                  (TG/VK/MAX/Позвоните)
                                     │
                                     ▼
                                  Webhook → Google Sheet
                                     │
                                     ├── Лид записывается
                                     ├── Партнёру уведомление
                                     └── amoCRM (позже)
```

## Компоненты системы

### 1. Google Sheet «ДОНУЛЯ Партнёрка»

#### Лист «Партнёры»
| Столбец | Описание |
|---------|----------|
| ID | Авто-номер |
| Имя | Имя партнёра |
| Телефон | +7... |
| Ref-код | Уникальный код (напр. IVAN123) |
| Мессенджер | TG / VK / MAX |
| ID в мессенджере | chat_id для уведомлений |
| Дата регистрации | Автоматически |
| Статус | Активный / Неактивный |

#### Лист «Лиды»
| Столбец | Описание |
|---------|----------|
| ID | Авто-номер |
| Имя клиента | Из формы |
| Телефон клиента | Из формы |
| Сумма долга | Из формы (250-500, 500-1000, и т.д.) |
| Мессенджер клиента | TG / VK / MAX / Телефон |
| Ref-код | Код партнёра из URL |
| Канал | «ссылка» или «прямой» |
| Статус | Новый → В работе → Консультация → Договор → Выплата |
| Дата | Автоматически |
| Юрист | Кто взял в работу |

#### Лист «Дашборд»
- Сводная по партнёрам: лидов / договоров / сумма к выплате
- Формулы COUNTIF/SUMIF
- Автообновление

### 2. Apps Script (в Google Sheet)

**doPost(e)** — принимает POST от ref.html:
```
Вход: {name, phone, debt, messenger, ref, timestamp}
Действия:
  1. Записать в лист «Лиды»
  2. Найти партнёра по ref-коду
  3. Отправить email партнёру: "От вас пришёл лид: Иван, юрист начал работу"
  4. (Позже) Создать сделку в amoCRM
```

### 3. ref.html — форма с webhook

Текущий `console.log` заменяется на:
```javascript
fetch('https://script.google.com/macros/s/DEPLOY_ID/exec', {
  method: 'POST',
  body: JSON.stringify(data)
})
```

### 4. Бот в мессенджерах (следующий этап)

При регистрации партнёра:
- Бот присылает приветствие + реф. ссылку
- Бот принимает номера: партнёр пишет "Иван +79121234567"
- Бот парсит → добавляет в Google Sheet → уведомляет

При новом лиде по ссылке:
- Бот пишет партнёру: "🎉 От вас новый лид: Иван. Юрист начал работу!"

## Генерация ref-кодов

Простая схема: первые 4 буквы имени в транслите + 3 случайные цифры
- Иван → IVAN482
- Сергей → SERG117
- Ольга → OLGA935

Генерируется автоматически при добавлении партнёра в таблицу.

## Путь пользователя (User Journey)

### Партнёр
1. Заходит на v2.html → проходит обучение (15 экранов)
2. Нажимает "Начать зарабатывать и перейти в группу"
3. Попадает в мессенджер (TG/VK/MAX)
4. Бот/оператор присваивает ref-код, партнёр получает ссылку
5. Партнёр раскидывает ссылку в чаты, группы, соцсети
6. Получает уведомления о новых лидах
7. Когда клиент заключает договор → партнёр получает 10 000 ₽

### Клиент (по реф. ссылке)
1. Видит ссылку в чате/группе от партнёра
2. Переходит на ref.html → видит лендинг
3. Заполняет форму: имя, телефон, сумма долга
4. Выбирает мессенджер для связи
5. Заявка автоматически попадает в систему
6. Юрист связывается в течение 30 минут

### Клиент (прямой контакт)
1. Партнёр лично рекомендует
2. Партнёр отправляет имя + номер боту/в группу
3. Данные попадают в Google Sheet
4. Юрист связывается

## Бот-ЛК партнёра (Telegram / VK / MAX)

Бот = личный кабинет. Партнёр делает всё через мессенджер, не заходя на сайт.

### Регистрация (после обучения)
```
Партнёр нажимает "Начать зарабатывать и перейти в группу"
    → Попадает в бота
    → Бот: "Привет! Как вас зовут?"
    → Партнёр: "Иван"
    → Бот: "Ваш номер телефона?"
    → Партнёр: "+79121234567"
    → Бот генерирует ref-код (IVAN482)
    → Бот записывает в Google Sheet лист "Партнёры"
    → Бот: "Готово! Ваша реферальная ссылка:
            donulya.ru/ref?ref=IVAN482

            Делитесь ей в чатах и соцсетях —
            за каждый договор вы получаете 10 000 ₽"
```

### Команды бота

| Команда | Что делает |
|---------|-----------|
| /start | Регистрация нового партнёра |
| /link или «Моя ссылка» | Показать реферальную ссылку |
| /leads или «Мои лиды» | Список лидов с статусами |
| /balance или «Мой баланс» | Сколько заработано / к выплате |
| /send или «Передать номер» | Отправить контакт клиента напрямую |
| /help | Справка по командам |

### Передача номера через бота
```
Партнёр: /send
Бот: "Введите имя и телефон клиента"
Партнёр: "Сергей +79031234567"
Бот: "✅ Принято! Лид 'Сергей' добавлен.
      Юрист свяжется с ним в течение 30 минут.
      Статус можно отследить: /leads"
    → Записывает в Google Sheet
    → (Позже) Создаёт сделку в amoCRM
```

### Техническая реализация

**Стек:** Node.js + telegraf (TG) / vk-io (VK) / MAX API
**Данные:** Google Sheets API (чтение/запись)
**Хостинг:** тот же сервер или Vercel/Railway (бесплатно)
**Деплой:** webhook-режим (не polling)

Один бэкенд обслуживает все 3 мессенджера — логика общая, адаптеры разные.

## amoCRM — структура с нуля

### Воронка «Партнёрские лиды»

| Этап | Цвет | Описание |
|------|------|----------|
| Новый лид | 🔵 | Заявка с ref.html или от партнёра напрямую |
| Квалификация | 🟡 | Юрист проверяет: долг ≥250к, подходит ли |
| Консультация назначена | 🟠 | Звонок/встреча запланирована |
| Консультация проведена | 🟣 | Юрист провёл, клиент думает |
| Договор заключён | 🟢 | Клиент подписал → партнёру +10 000 ₽ |
| Выплата партнёру | ✅ | Вознаграждение выплачено |
| Отказ | 🔴 | Клиент отказался / не подходит |

### Воронка «Партнёры»

| Этап | Описание |
|------|----------|
| Новый партнёр | Прошёл обучение, получил ref-код |
| Активный | Передаёт лиды |
| Топ-партнёр | 5+ договоров |
| Неактивный | Нет лидов 30+ дней |

### Пользовательские поля (сделка «Партнёрские лиды»)

| Поле | Тип | Описание |
|------|-----|----------|
| Ref-код партнёра | Текст | IVAN123 |
| Имя партнёра | Текст | Автоподстановка по ref-коду |
| Канал привлечения | Список | Ссылка / Прямой контакт |
| Сумма долга | Список | 250-500к / 500к-1М / 1-3М / 3М+ |
| Мессенджер клиента | Список | TG / VK / MAX / Телефон |
| Статус выплаты | Список | Не выплачено / Выплачено |
| Сумма выплаты | Число | 10000 |

### Пользовательские поля (контакт «Партнёр»)

| Поле | Тип | Описание |
|------|-----|----------|
| Ref-код | Текст | Уникальный код партнёра |
| Мессенджер | Список | TG / VK / MAX |
| ID в мессенджере | Текст | Для уведомлений |
| Всего лидов | Число | Автосчёт |
| Всего договоров | Число | Автосчёт |
| Сумма к выплате | Число | Автосчёт |

### Автоматизации в amoCRM (Sensei или Digital Pipeline)

1. **Новый лид** → автозадача юристу «Позвонить клиенту за 30 мин»
2. **Договор заключён** → уведомление партнёру + изменить статус выплаты
3. **Нет активности 2 дня** → напоминание юристу
4. **Отказ** → уведомление партнёру «Клиент отказался»

### Каналы связи в amoCRM

| Канал | Интеграция | Для чего |
|-------|-----------|----------|
| Телефония | Мегафон / МТТ / Sipuni / OKI-TOKI | Звонки юристов клиентам, запись разговоров |
| Telegram | Wazzup / Radist.Online / amoCRM встроенный | Переписка с клиентами и партнёрами |
| ВКонтакте | amoCRM встроенный / Radist.Online | Сообщения сообщества |
| MAX (VK Teams) | Через API / webhook | Если появится интеграция |
| WhatsApp | Wazzup | Опционально |

**Что это даёт:**
- Юрист звонит клиенту прямо из amoCRM — звонок записывается в сделку
- Переписка из TG/VK видна в карточке сделки
- История коммуникации сохраняется
- Можно слушать звонки для контроля качества

### Интеграция Google Sheet → amoCRM

```
ref.html форма
    → Apps Script webhook (Google Sheet)
    → Apps Script вызывает amoCRM API
    → Создаёт сделку в воронке «Партнёрские лиды»
    → Этап «Новый лид»
    → Заполняет все поля (ref-код, канал, долг, мессенджер)
    → Создаёт контакт клиента
    → Привязывает к партнёру
```
